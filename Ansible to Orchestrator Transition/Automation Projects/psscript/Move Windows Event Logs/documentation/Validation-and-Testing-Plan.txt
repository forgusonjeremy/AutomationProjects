A6. Confirm service account — UNC read access
    Run from PS host under the service account context:
      Test-Path '\\targetserver\C$\Windows\System32\winevt\Logs'
    Expected: True
    If False: Review PS host service account permissions on target servers.

A7. Confirm service account — UNC write access
    Run from PS host under the service account context:
      New-Item -Path '\\fileserver\mdcarchivelog$\Windows\test-write-check' -ItemType Directory -Force

    Expected: Both commands succeed without errors.

A8. Confirm Kerberos constrained delegation (double-hop)
    The PS host reads from \\targetserver\C$\... and writes to \\fileserver\share$\...
    This is a double-hop. The PS host machine account or service account must be
    trusted for constrained delegation to the file server SPN.
    Check: AD Users and Computers → PS host account → Delegation tab
      [ ] "Trust this computer for delegation to specified services only" is set
      [ ] File server CIFS SPN is listed in the allowed services
    If not configured: Engage AD team before running live workflows.

A9. Confirm RSAT AD Tools on PS host
    Run on PS host:
      Get-Module -ListAvailable ActiveDirectory
    Expected: Module is listed.
    If not found: Install RSAT on PS host before running ByADGroupName or ByADGroupCN workflows.

ROLLBACK AND RECOVERY
───────────────────────────────────────────────────────────────────────────────

Item                     Rollback Approach
─────────────────────── ────────────────────────────────────────────────────────────────────────────────────────
cvs_functions.ps1        Keep a backup of the original before applying additive changes.
                         Restore from backup if the new switch case causes regression.
                         The change is isolated to a new switch case; existing cases are unmodified.

Moved .evtx files        Files are moved, not deleted.  If a move operation places files at the wrong
                         destination, they can be manually moved back.
                         The script creates per-server subfolders; identify the correct destination by
                         matching server short name.

Deleted files (Remove)   No automated recovery — deletion is permanent.
                         Mitigation: Always run with whatIf=yes first to preview scope.
                         For production: confirm output of whatIf=yes run before approving whatIf=no.

vRO content              All items are new (no modifications to existing vRO content).
                         Rollback = delete the four workflows, five actions, and Config Element.
                         The OOTB "Invoke a PowerShell script" workflow is not modified.

Config Element           Delete or update attribute values to revert defaults.
                         No production impact until workflows are run.
